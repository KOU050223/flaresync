# ペイロードサイズ計測 — JSON vs MessagePack

計測日: 2026-04-22  
実装: `msgpackr@1.11.10`  
スクリプト: [`scripts/measure-payload.ts`](../scripts/measure-payload.ts)

## 結果

| パターン | JSON | MessagePack | 削減率 |
|---|---|---|---|
| フラットな単一キー (`hp: 90`) | 33 bytes | 26 bytes | 21.2% |
| フラットな複数キー (3つ) | 46 bytes | 32 bytes | 30.4% |
| ネストしたパス (`player.x`, `player.y`) | 53 bytes | 42 bytes | 20.8% |
| Map.set() 単一操作 | 81 bytes | 62 bytes | 23.5% |
| Map.set() 複数操作 (3件) | 157 bytes | 115 bytes | 26.8% |
| 大きめのゲームステート | 191 bytes | 134 bytes | 29.8% |

**平均削減率: 25.4%**

## 所感

- 小さいペイロードでも約 20% の削減が得られる
- ペイロードが大きくなるほど削減率が上がる傾向（キー名の繰り返しが減るため）
- ゲームのように高頻度で小さなパッチを送るユースケースでは、絶対バイト数より**メッセージ数**の削減（50ms バッチング）の方がインパクトが大きい
- バイナリ通信への切り替えは追加コストゼロで 25% 程度の通信量削減が見込める
